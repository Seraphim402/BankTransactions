# Отчет: банковские транзакции

## Выбранные технологии

- Язык: C#.
- База данных: SQLite.
- Сервер: консольное приложение `BankTransactions.Server`.
- Клиент: оконное WPF-приложение `BankTransactions.Client`.
- Тесты: xUnit, проект `BankTransactions.Tests`.

## Архитектура

Приложение состоит из трех основных частей:

- `BankTransactions.Core` - общая библиотека с моделями, созданием БД и бизнес-логикой.
- `BankTransactions.Server` - консольный TCP-сервер. Он принимает подключения от нескольких оконных клиентов и выводит количество подключенных клиентов.
- `BankTransactions.Client` - графическое окно для входа, создания клиентов, счетов и проведения транзакций.

Клиент не обращается к SQLite напрямую. Все операции проходят через сервер.

## Таблицы базы данных

- `Users` - пользователи системы. Начальный пользователь: `admin/admin`.
- `Clients` - клиенты банка: физические и юридические лица.
- `Accounts` - счета клиентов: зарплатный, валютный, накопительный.
- `PartnerBanks` - банки-партнеры, включая зарубежные.
- `Commissions` - проценты комиссий по типам счетов и признаку зарубежного партнера.
- `Transactions` - история пополнений и переводов.

## Правила операций

- Для создания, удаления клиента, создания счета и проведения транзакции требуется вход.
- У клиента может быть несколько счетов.
- При переводе сервер проверяет баланс счета списания.
- Комиссия рассчитывается по типу счета и типу банка-партнера.
- С баланса отправителя списывается сумма перевода и комиссия.
- На счет получателя зачисляется только сумма перевода.

## Сценарий демонстрации

1. Запустить консольный сервер.
2. Запустить два окна клиента и показать, что сервер отображает количество подключенных клиентов.
3. В одном клиенте выполнить вход `admin/admin`.
4. Создать клиента-физическое лицо.
5. Создать клиента-юридическое лицо.
6. Создать зарплатный счет для физического лица.
7. Создать валютный счет для юридического лица.
8. Пополнить зарплатный счет на 1000.
9. Выполнить перевод 100 через зарубежный банк-партнер.
10. Открыть вкладку переводов и показать запись транзакции с комиссией.
11. Удалить клиента после входа в систему.

## Unit-тесты

В проекте `BankTransactions.Tests` есть проверки:

- вход под начальным администратором;
- пополнение счета;
- перевод с расчетом комиссии;
- запрет перевода при недостаточном балансе.
